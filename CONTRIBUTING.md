# Contributing to RAG Architecture Learning Resource

Thank you for your interest in contributing!

## How to Contribute

### 1. Adding New Content

- **New Topics**: Create a new markdown file in the appropriate section
- **Code Examples**: Ensure code is tested and includes comments
- **Notebooks**: Follow the existing notebook structure

### 2. Documentation Standards

- Use clear, concise language
- Include code examples where appropriate
- Add diagrams using ASCII art or Mermaid
- Reference research papers where applicable

### 3. Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-topic`)
3. Make your changes
4. Commit with descriptive messages
5. Push to your fork
6. Open a Pull Request

## Content Guidelines

### Theory Documents
- Explain concepts clearly
- Include visual diagrams
- Reference academic papers

### Architecture Documents
- Include implementation code
- Explain pros/cons
- Provide use case recommendations

### Notebooks
- Include installation instructions
- Add comments in code
- Test all code before committing

## Questions?

Open an issue for:
- Questions about content
- Suggesting new topics
- Reporting errors
